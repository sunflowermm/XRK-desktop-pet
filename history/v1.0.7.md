## v1.0.7

### 亮点
- **更新弹窗 Markdown 渲染增强**：支持更多常用语法（标题/有序列表/链接/斜体/分隔线），并补齐转义与链接安全校验，展示更稳定。
- **Gitee 更新下载更可靠**：安装包产物改为英文文件名模板，避免中文 URL 编码导致下载链接异常/被截断。
- **右键菜单体验优化**：单窗口复用 + 启动预创建，无加载圆环、无“只弹出一次”；窗口与页面透明，仅菜单块有背景，子菜单无多余色块；resize 后立即显示，减少闪烁。

### 更新
- **更新弹窗（update-dialog）**：
  - Markdown：支持 `#`~`####`、`1.` 有序列表、`[text](url)` 链接、`*斜体*`、`---` 分隔线
  - 安全：文本统一 HTML 转义；链接仅允许 `http(s)`/`mailto`，并过滤 href 中危险字符
  - 样式：补齐 h5/h6、ol、a 的基础样式
- **右键菜单（menu）**：
  - 主进程：菜单窗口复用，关闭时 hide 不 destroy；启动时预创建并加载，首次右键即用；`show-menu-window` 仅传 clientX/clientY，由主进程算屏幕坐标；统一 `sendMenuPosition(win)` 下发位置与屏幕 bounds，移除 `get-screen-bounds` IPC
  - 渲染：body 透明，仅 `#menu`、`.submenu` 有深色背景与 backdrop-filter；子菜单 `width: max-content`，无整块无意义背景；通过 `menu-set-data` 更新数据并重绘，依赖 `resize-menu-window` 触发显示（不依赖 rAF，避免隐藏窗口节流导致不弹出）
  - 显示逻辑：resize 后立即 `show()`，去掉 setImmediate，减轻闪烁；blur 延迟 80ms 且仅对可见窗口 hide

### 修复
- **更新模块（updater）**：清理未使用 IPC 与冗余导出，托盘「清理更新缓存」调用链统一，减少维护成本。
- **右键菜单**：修复隐藏窗口下 requestAnimationFrame 不触发导致 `menu-ready` 永不到达、菜单只成功弹出一次的问题；去掉对 `menu-ready` 的依赖，改为 resize 即显示。

### 说明
- 本次为功能与稳定性更新，建议所有用户升级。

### 更新机制（是否必须下载完整包）
- **不必一定下完整包**。electron-updater 支持**差分更新**：当服务器提供与 `latest.yml` 中 path 对应的 `.exe.blockmap` 时，会先尝试只下载相对当前版本变化的块（differential），失败或不可用时再回退为完整安装包下载。
- **当前项目**：electron-builder 对 NSIS 会默认生成 `SunflowerDesktopPet-Setup-${version}.exe.blockmap`，发布脚本已将 `latest.yml`、`.exe`、`.blockmap` 一并上传，因此已具备差分更新条件。

**为何每次更新都是完整包（常见原因）**  
差分更新需要同时满足以下条件，否则会回退为完整包下载：
1. **当前版本的安装包已被缓存**：差分是与“当前已安装版本”的安装包做块对比。若用户是首次通过安装程序安装（未经过应用内更新），或曾点过「清理更新缓存」，则没有可用的旧安装包/blockmap，只能完整下载。
2. **服务器支持 HTTP Range**：Gitee 的 Release 附件下载若未返回 `Accept-Ranges: bytes`，差分请求会失败并回退完整包。可对附件 URL 发 HEAD 请求查看响应头确认。
3. **Blockmap 可访问且可解析**：若 `.exe.blockmap` 下载失败、被代理/重写改坏，或校验不通过（如 SHA512 不一致），会报 “Cannot parse blockmap” / “sha512 checksum mismatch” 并回退完整包。

**排查建议**：在设置中开启「调试模式」后，更新时可在控制台看到 `updater-info` / `updater-warn` / `updater-error`，其中会有 “Cannot download differentially, fallback to full download” 及具体原因（如缺少缓存、Range 不支持、blockmap 错误等）。

- **其他**：NSIS 强压缩会降低块相似度，差分效果可能有限；若需更好差分率可在 `nsis` 中尝试 `compression: 'store'`（安装包会变大）。若希望强制始终完整包可设 `autoUpdater.disableDifferentialDownload = true`。
- 参考：electron-builder / electron-updater 官方文档与 [AppUpdater](https://www.electron.build/electron-updater.Class.AppUpdater.html)。
